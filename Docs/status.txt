# Overall Summary

## Overall project status

**Planning complete / implementation not started.**

MR Guardian is now defined as a Python-based Unity Merge Request review assistant with a CLI, Streamlit dashboard, YAML executable policy, optional human guidance, stable rule IDs, deterministic Unity-specific checks, persistent SQLite review history, and optional AI summaries.

The project is positioned as a realistic portfolio piece because it demonstrates agentic coding, policy-as-code, Unity engineering judgment, safe AI-assisted development practices, and workflow analytics for project managers.

## Main implementation priorities

1. Build the Python core and local CLI.
2. Implement YAML policy loading with deterministic and LLM rule types.
3. Add SQLite review history.
4. Add Streamlit dashboard views.
5. Implement Unity scene, prefab, ProjectSettings, metadata, C#, test, and AI-code risk rules.
6. Add review reporting and suggested fixes.
7. Add optional AI summary.
8. Add GitLab integration after the local MVP works.

## Current gaps

- Repository has not been created yet.
- No Python package implementation exists yet.
- No YAML policy loader exists yet.
- No external rule ID document validator is needed; YAML is the only runtime policy source.
- No rule engine exists yet.
- No SQLite review history exists yet.
- No Streamlit dashboard exists yet.
- No demo fixtures exist yet.
- No GitLab integration exists yet.
- No optional AI summary exists yet.
